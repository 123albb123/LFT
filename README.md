# 内网文件传输工具

一款免安装、按需运行的 Windows 局域网文件传输工具。打开软件即启动本地 HTTP 服务，退出软件后服务自动停止并释放端口。

## 主要功能

- Windows 桌面端管理共享文件，支持拖放添加、右键复制链接、二维码、编辑、删除和打开目录。
- 手机与电脑均可通过 Web 页面上传、下载和查看文件列表。
- 支持中文文件名、大文件流式传输、上传与下载进度、Range 断点续传。
- 文件内容修改后直链保持不变，桌面端与 Web 列表自动刷新。
- 默认仅允许同一局域网访问，Web 删除默认关闭。
- 所有数据、配置和日志默认保存在程序目录，不安装服务，不设置开机启动。

## 快速开始

1. 从 [Releases](https://github.com/123albb123/LFT/releases) 下载 `LanFileTransfer-win-x64.zip`。
2. 解压到具有写入权限的普通目录，不要直接在压缩包内运行。
3. 双击 `内网文件传输工具.exe`。
4. Windows 防火墙首次询问时，如需其他设备访问，只允许“专用网络”。
5. 其他设备打开软件显示的地址，例如 `http://192.168.1.88:28080/web`。

文件直链示例：

```text
http://192.168.1.88:28080/xx.conf
```

完整操作和故障排查请查看 [使用说明](docs/USER-GUIDE.zh-CN.md)。

## 默认设置

| 设置 | 默认值 |
| --- | --- |
| 端口 | `28080` |
| 仅允许局域网访问 | 开启 |
| Web 上传 | 开启 |
| Web 删除 | 关闭 |
| 同名文件处理 | 覆盖 |
| 单文件上传上限 | 2 GB |

## 目录说明

首次运行后，程序所在目录会自动创建：

```text
Data/                 共享和上传文件
Config/settings.json  软件配置
Logs/                 运行日志
```

如果在设置中选择了外部共享目录，文件将保存到用户指定的位置。

## 从源码构建

需要 Windows 10/11 x64 和 .NET 10 SDK：

```powershell
dotnet test .\LanFileTransfer.sln -c Release
.\build-package.ps1
```

免安装包生成到 `artifacts/LanFileTransfer-win-x64.zip`。

## 安全提示

同一局域网中的设备无需账号即可访问共享内容。请勿在公共或不受信任的网络中运行，也不要共享敏感文件。软件不会自动修改 Windows 防火墙规则。
